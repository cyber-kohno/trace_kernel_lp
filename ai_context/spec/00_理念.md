# 理念 — Programming as a Tool

多くのエンジニアにとって、プログラムは「アプリを作るためのもの」です。

そのため、
・ログを分析する
・データを加工する
・ファイルを整理する

そうした小さな問題に対して、わざわざ「プログラムを書こう」という発想になりません。
書くことが難しいのではありません。**実行できる状態にするまでが重い**のです。

・環境構築
・依存関係（npm等）
・設定

その準備が、プログラミングを“特別な行為”にしてしまいました。

電卓を使うとき、環境構築はしません。
メモを書くとき、プロジェクトは作りません。

道具とは、本来そういうものです。

もし、TypeScriptが、アプリを開いた瞬間に書いて、その場で実行できるとしたら。
必要なリソースが整理され、すぐに参照することができるとしたら。

プログラミングは、もっと身近な道具になるはずです。

## 万能な環境を目指さない「ハブ」としての役割

Trace Kernelは「何でもできる万能な開発環境」を目指していません。
そのため、npmなどの外部ライブラリの導入や、複雑なファイル分割（インポート等）はできない仕様としています。

数千行に及ぶような複雑な処理や、特定のサードパーティライブラリが必要な場合は、好きな別言語で実装し、Trace Kernelの外部実行機能（`$process`）から呼び出して「ハブ（テキストやパスを繋ぐ糊の役割）」として連携させることを推奨しています。

プログラムは、開発のための手段にとどまらない。
それは本来、問題を解くための道具である。

**Trace Kernelは、その当たり前を取り戻すために設計された実行基盤です。**
