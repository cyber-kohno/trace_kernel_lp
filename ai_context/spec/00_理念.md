# Philosophy — Programming as a Tool

多くのエンジニアにとって、  
プログラムは「アプリを作るためのもの」です。

そのため、  
・ログを分析する  
・データを加工する  
・ファイルを整理する  

そうした小さな問題に対して、  
わざわざ「プログラムを書こう」という発想になりません。

書くことが難しいのではありません。  
実行できる状態にするまでが重いのです。

・環境構築
・依存関係  
・設定

その準備が、  
プログラミングを“特別な行為”にしてしまいました。

電卓を使うとき、環境構築はしません。  
メモを書くとき、プロジェクトは作りません。

道具とは、本来そういうものです。

もし、TypeScriptが、
アプリ開いた瞬間に書いて、その場で実行できるとしたら。

必要なリソースが整理され、
すぐに参照することができるとしたら。

プログラミングは、  
もっと身近な道具になるはずです。


プログラムは、開発のための手段にとどまらない。  
それは本来、問題を解くための道具である。  

Trace Kernelは、  
その当たり前を取り戻すために設計された実行基盤です。
