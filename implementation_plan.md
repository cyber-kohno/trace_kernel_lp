# Trace Kernel LP 方向性と構成案

Trace Kernelのランディングページ（メインの無料版向け）の方向性と構成の初期案です。エンジニアが具体的な利点をイメージでき、「魔法の杖」的な誇大表現を避ける方針で策定しています。

## 1. ターゲットとトーン＆マナー

- **ターゲット**: TypeScriptを実用レベルで扱うエンジニア、またはプログラミング学習者。
- **トーン＆マナー**: 
  - 過度な修飾や煽り文句（「誰でも簡単」「1秒で解決」等）は極力排除。
  - 「環境構築なしでTSを実行できる」という事実に対し、「具体的にどういう仕組みで、何ができるのか」を技術的に誠実に伝える。
  - プロ向けツールの洗練された印象（例：Vercel, Stripe, Supabaseなどのような、余白を活かしたダークモード系のモダンデザイン）を想定。

## 2. LPの構成案（セクション案）

LPの流れとして以下の5つのセクションを提案します。

1. **Hero（ファーストビュー）**
   - **目的**: 何のアプリか一瞬で伝え、ダウンロードを促す。
   - **要素**:
     - メインコピー案：「環境構築という壁をなくした、TypeScript実行基盤」
     - サブコピー案：「プロジェクト作成も、npm installも不要。思いついた処理を、今すぐ型安全に実行できるエンジニアのためのサンドボックス。」
     - アクション：無料ダウンロードボタン
     - メインビジュアル：アプリの画面（コードエディタと実行結果画面）のスクリーンショット、または操作中のGIF。

2. **課題と解決（Why Trace Kernel?）**
   - **目的**: ターゲットの共感を得る。
   - **要素**:
     - 「日常のちょっとしたログ解析やデータ変換のために、わざわざプロジェクトフォルダを作り、開発環境を整えるのは重すぎる。」という課題感の提示。
     - それに対し、「Trace Kernelならアプリを開くだけで即座にTSが書ける」という解決策の提示。

3. **コア機能の紹介（How it works?）**
   - **目的**: どうやって実現しているのか、何が便利なのかを具体的に紹介する。
   - **要素**:
     - **コンテキスト一体型基盤**: 変数やリソースをGUIでセットアップし、ボイラープレート不要で即座に実行可能。
     - **強力なTypeScript補完**: GUIでの設定が自動注入され、型安全かつ快適なコーディングを実現。
     - **強力なリソース管理 (`resource`, `dataset`)**: 
       - `resource`: GUIに貼り付けるだけで、CSVなどがオブジェクト（JSON）としてパースされプログラムから扱える。
       - `dataset`: 面倒なローカルファイルやディレクトリの走査・フィルタリングをGUIの設定のみで実現。

4. **クリーンな環境（No Clutter）**
   - **目的**: 導入の心理的ハードルを下げる。
   - **要素**:
     - 物理ファイル（package.jsonやコンパイル後のjs）が裏で生成されてローカル環境を散らかすことはない。
     - すべてのコードと状態はワークスペースファイル（`.trk`）内で完結。

5. **CTA（コールトゥアクション）**
   - **目的**: 読後の再度のダウンロード誘導。
   - **要素**: 
     - 「今すぐ無料ではじめる」といったダウンロードボタンへの導線。

※ ご要望の通り、有料版機能にあたる `$process`, `$fs` APIに関する言及や、有料版への誘導はこのページには一切含めません。

## 3. ユーザーへの確認事項（質問）

> [!IMPORTANT]
> 方向性を確定させるために、以下の点についてご意見をお聞かせください。

1. **デザインの雰囲気について**：
   上記で挙げたような「ダークモード基調のモダンな開発者向けデザイン（例：Vite公式やVercelなど）」を想定していますが、ご希望に近いサイトのイメージ等はございますでしょうか？（もしくはライトモードをご希望でしょうか？）

2. **メインビジュアル・素材について**：
   エンジニア向けLPでは、実際の機能がどう動いているかがわかるスクリーンショットやGIF動画（補完が効いている様子や実行結果など）があると非常に説得力が増します。後ほど実装時で構いませんので、こうした画像素材を用意していただくことは可能でしょうか？

3. **ユースケースのコード例について**：
   LP内に「例えばこんな処理が数行で書けます」という具体的な短いコード例を載せると利用イメージが伝わりやすいです。Trace Kernelの良さが伝わるようなコード例（`$resource.xxx`を使っているものなど）のアイデアがあれば教えてください。
